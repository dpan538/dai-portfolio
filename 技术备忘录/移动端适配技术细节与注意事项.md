# 移动端适配技术细节与注意事项

## 一、断点与视口

### 1.1 统一断点
- **主断点：768px**  
  - `max-width: 768px`：视为移动端  
  - `min-width: 769px`：视为桌面端  
- **次要断点**：各页面可能有 720px、800px、820px、860px、900px 等，用于响应式微调

### 1.2 移动端视口
- `main.content` 在移动端：`padding-top: 142px`（为固定 header 留空）
- `padding-inline: 26px`（`--mobile-padding-x`）
- `padding-bottom: max(48px, env(safe-area-inset-bottom, 0px))`

---

## 二、布局策略

### 2.1 桌面/移动端分离策略
部分项目采用「同一页面内双版」或「独立移动端页面」：

| 页面 | 策略 | 说明 |
|------|------|------|
| **Quite Off** | 独立页面 | `quite_off.astro` 仅桌面，`quite_off_mobile.astro` 仅移动，通过 JS 按视口重定向 |
| **Reframed Still** | 同页双版 | `.reframed-desktop` 与 `.reframed-mobile` 用 `@media` 切换显示 |
| **Masks** | 同页双版 | `.masks-desktop` 与 `.masks-mobile` 切换 |
| **Closing Time** | 响应式 | 同一结构，用 `@media` 调整布局与间距 |
| **About** | 响应式 + 内容差异 | 移动端显示精简介绍、Instagram @，桌面端显示完整内容 |

### 2.2 ProjectMobileLayout
- **路径**：`src/layouts/project_mobile.astro`
- **作用**：提供统一移动端容器样式  
  - `padding-top: 142px`  
  - `padding-inline: var(--mobile-padding-x, 26px)`  
  - `padding-bottom: 48px`  
- **使用场景**：`quite_off_mobile.astro` 等需要固定顶部留白的移动端页面

---

## 三、滚动与滚动条

### 3.1 Lenis 平滑滚动
- **脚本**：`src/scripts/lenis-mobile.js`（由 `base_layout.astro` 引入）
- **触发条件**：`window.innerWidth <= 768` 时初始化
- **效果**：在 body/html 上启用平滑滚动
- **事件**：`lenis:pause` / `lenis:resume` 可暂停/恢复（如 lightbox 打开时）

### 3.2 滚动条隐藏
- **全局**：`base_layout.astro` 中 `@media (max-width: 768px)` 下对 `html`、`body` 设置：
  - `::-webkit-scrollbar { display: none; width: 0; }`
  - `scrollbar-width: none`
  - `-ms-overflow-style: none`
- **Reframed Still**：额外对 `main.content:has(.reframed-mobile)` 做相同隐藏（早期实现，现由全局规则覆盖）

### 3.3 横向滚动
- **Reframed Still**：`.strip` 为横向滚动，`overflow-x: auto`，滚动条已隐藏
- **禁止整页横向滚动**：相关页面设置 `overflow-x: hidden`，仅图片 strip 可左右滑动

---

## 四、Quite Off 特殊逻辑

### 4.1 路由重定向
- 访问 `/quite_off` 且 `window.innerWidth <= 768` → 重定向到 `/quite_off_mobile`
- 访问 `/quite_off_mobile` 且 `window.innerWidth > 768` → 重定向到 `/quite_off`
- **实现**：各页面内 `<script is:inline>` 检测视口并 `window.location.replace()`

### 4.2 QuiteOffMobile 组件
- **路径**：`src/components/projects/QuiteOffMobile.astro`
- **结构**：图片与段落交替（img0, p1, img1, img2, p2, …）
- **图片**：`width: 100%`、`max-width: 100%`，避免横向溢出

---

## 五、Shell 与内容高度

### 5.1 Reframed Still 的 Shell 高度
- 桌面端：`min-height: 100vh`（固定高度）
- Reframed Still 移动端：`body:has(main.content:has(.reframed-mobile)) .shell` 设置 `min-height: auto`
- **目的**：内容较少时，页面不强行撑满视口，减少底部空白

---

## 六、菜单与交互

### 6.1 移动端 Header
- **位置**：`base_layout.astro` 中 `.mobile_header`
- **显示条件**：`@media (max-width: 768px)` 时 `display: flex`
- **内容**：品牌名 + 汉堡菜单按钮

### 6.2 移动端全屏菜单
- **ID**：`mobile_overlay`
- **打开**：body 加 `mobile-overlay-open`，滚动锁定
- **关闭**：ESC、点击 backdrop 或关闭按钮

### 6.3 取消按钮尺寸
- 关闭图标（✕）：`font-size: 20px`，与汉堡菜单视觉平衡

---

## 七、注意事项

### 7.1 修改范围
- 明确「仅移动端」时，所有样式与逻辑必须放在 `@media (max-width: 768px)` 内，或通过 `.reframed-mobile`、`.masks-mobile` 等移动专用 class 限定

### 7.2 文字对齐
- 描述区域：`text-align: left`，与 Dai Pan 左边缘对齐（依赖 `main.content` 的 `padding-inline`）

### 7.3 图片与描述间距
- 统一使用 `12px` 作为图片与描述之间的垂直间距，与其他项目页一致

### 7.4 TypeScript / 内联脚本
- Astro 内联 `<script>` 可能被当作 JS 解析，TypeScript 语法易报错
- 建议：使用 `// @ts-nocheck` 关闭该块检查，或将逻辑抽到独立 `.ts` 文件再导入

### 7.5 测试
- 在 320×736、375×667 等移动视口下验证布局与滚动
- 确保横向无整页滚动，仅有设计中的横向滚动区域（如 strip）可滑动
